# Store Manager App - QR Code Tracking & POS Integration Plan

## Overview
This document outlines the integration of QR code inventory tracking and point-of-sale (POS) system into the existing Store Manager App. The app is a multi-tenant platform serving multiple business tenants with multiple locations per business.

**Note:** Multi-tenant architecture and location switching are already implemented and not included in this plan.

---

## 1. User Profile & Scanner Setup

### 1.1 Scanner Setup Interface
- Access through **User Profile** → **"Setup Scanner"** option
- Displays a **dynamic QR code** (new code generated each time the Setup Scanner page is accessed)
- User scans this QR code using the Scanner App on their phone to establish the link

### 1.2 Scanner Linking Rules
- **One scanner per user only** - A user cannot have multiple scanners linked to their account
- **Permanent login** - Scanner stays logged in until the user manually logs out
- **Security** - The dynamic QR code prevents unauthorized scanner linking

## 2. Inventory Management - QR Code Tracking

### 2.1 Entry Point
1. Navigate to **Inventory** section
2. Select item from inventory list (using available filters: branch, category, etc.)
3. Click on **"Quantity available"** in the item row

### 2.2 Quantity Interface
Opens an interface displaying:
- **Total quantity** of the selected item
- **Old items** - Items already tracked with QR codes
- **New items** - Items without QR codes (need to be tracked)

### 2.3 QR Code Tracking Workflow

#### Step 1: Initiate Tracking
- Click **"Assign Tracking Codes"** button at the top of the interface
- System automatically filters to show only items **WITHOUT QR codes**

#### Step 2: For Each Untracked Item
Each untracked item displays:
- A **"Scan QR" icon**
- An **empty box** to display scanned QR code data

#### Step 3: Scan Process
1. Click on an untracked item → Sends command to Scanner App
2. Scanner App opens with **"Tracking Codes"** interface
3. User clicks **"Tracking Codes"** button
4. Camera opens automatically
5. User scans the **physical QR code** attached to the item
6. **Beep sound** confirms successful scan
7. Camera closes automatically
8. Returns to Scanner interface showing QR code value under **"Tracking Code"**
9. User clicks **"Save"** in Scanner
10. Scanner command closes
11. QR code value automatically appears in the box for that item in Store Manager App

### 2.4 Save Options

#### Individual Save
- Each item box has a **"Save" button**
- User can save each tracked item individually
- Allows pausing and resuming tracking later

#### Batch Save
- **General "Save" button** at the end of the form (in app theme color)
- Saves all scanned items at once
- **Partial save capability:** If some items are scanned and others are not:
  - Only scanned items are saved
  - Unscanned items remain empty for later completion
- User can return later to complete unscanned items

## 3. Point of Sale (POS) System

### 3.1 POS Entry Point

1. Navigate to **Sales/POS** section
2. Choose one of two options:
   - **"New Sale"** - Single item transaction
   - **"Batch Sale"** - Multiple items transaction

### 3.2 Customer Entry (Common for Both Sale Types)

**This is the FIRST step for both New Sale and Batch Sale**

Opens a **Customer Entry Form** with two options:

#### Option A: New Customer
- Enter **customer name** (required)
- Enter **phone number** (required)
- Customer ID is auto-generated by the system

#### Option B: Existing Customer
- Search by **phone number**
- System displays matching customer(s)
- Select the customer from results

After selecting/entering customer details:
- Click **"Save"** or **"Confirm"**
- Sales form opens with customer details displayed (read-only)

---

## 4. NEW SALE Workflow (Single Item)

**Prerequisite:** Customer details have been entered/selected

### Step 1: Sales Form Opens
- Display customer details (name, phone, Customer ID)
- Ready to add item

### Step 2: Add Item
1. Click **"Add Item"** icon
2. Opens item search interface
3. Search by:
   - Model number
   - SKU
   - Item name
4. Select the desired item
5. Returns to Sales Form with item details displayed:
   - Item name
   - Model number
   - SKU

### Step 3: Scan QR Code
1. Click **"Scan QR Code"** icon
2. Creates scan command with popup notification
3. Scanner App automatically opens
4. User scans the physical QR code on the item
5. Upon successful scan:
   - Scanner interface displays QR code value
   - User saves in Scanner App
   - Popup automatically closes
6. QR code value appears in the QR code box in Sales Form

### Step 4: Complete Sale
1. Enter **amount paid** by customer
2. Click **"Complete Sale"** button
3. System proceeds to receipt generation

### Step 5: Receipt & Confirmation
1. Receipt is automatically generated
2. User can:
   - **Print** the receipt
   - **Save as PDF** on device
3. Confirmation screen is displayed
4. Sale is finalized

---

## 5. BATCH SALE Workflow (Multiple Items)

**Prerequisite:** Customer details have been entered/selected

### Step 1: Batch Form Opens
- Display customer details (name, phone, Customer ID)
- **Top action buttons:**
  - **"Add Items"** - To add items to batch
  - **"Delete Batch"** - To cancel entire batch (clears all items)

### Step 2: Add First Item
1. Click **"Add Items"** button at top
2. Opens item search interface
3. Search and select item by:
   - Model number
   - SKU
   - Item name
4. A box appears asking: **"How many items?"**
5. Enter quantity (e.g., 5, 35, etc.)
6. System automatically creates **that many QR scanner boxes** (one for each item)

### Step 3: Scan QR Codes for Each Item
- For each QR scanner box created:
  - Click the box → Scanner command is sent
  - Scanner App opens
  - User scans the individual item's QR code
  - QR code value appears in the box
  - User can save individually or continue to next item

### Step 4: Add More Items (Optional)
- Click **"Add Items"** button again
- Repeat Steps 2-3 for additional items
- Can add:
  - Multiple quantities of the same item
  - Different items in the same batch

### Step 5: Delete Batch (If Needed)
- At any time before receipt generation, click **"Delete Batch"**
- Entire batch is cleared
- All added items are removed
- User returns to empty batch form

### Step 6: Complete Sale
1. After all items are scanned and added
2. Click **"Complete Sale"** button
3. Enter **amount paid** by customer
4. System proceeds to receipt generation

### Step 7: Receipt & Confirmation
1. Receipt is automatically generated
2. User can:
   - **Print** the receipt
   - **Save as PDF** on device
3. Confirmation screen is displayed
4. Sale is finalized

## 6. Inventory Update - Two-Stage Process

### 6.1 TEMPORARY Update (When "Complete Sale" is Clicked)

**Timing:** Immediately when **"Complete Sale"** button is clicked

**What happens:**
- Sale is confirmed
- Inventory is **temporarily reduced** by the number of items sold
- Item is displayed in a **pending box** (visual indicator)
- Status shows as "Pending Sale"

**Why temporary:**
- Sale is not yet final (receipt not yet generated)
- Grace period for user to make changes

### 6.2 PERMANENT Update (When Receipt is Generated)

**Timing:** When **amount paid is entered and receipt is generated**

**What happens:**
- Inventory reduction becomes **permanent**
- Sale is fully finalized
- Item status changes from "Pending Sale" to "Sold"
- Data is permanently written to backend

### 6.3 Cancellation: Delete Batch (Before Receipt Generation)

**If user clicks "Delete Batch" before receipt generation:**
- Temporary inventory reduction is **reversed**
- Inventory quantity returns to **original level**
- Sale is **cancelled completely**
- No permanent record created

**Example:**
- Item had quantity of 100
- User clicks "Complete Sale" → Quantity temporarily becomes 95
- User clicks "Delete Batch" → Quantity returns to 100
- If receipt generated → Quantity permanently becomes 95

## 7. Receipt Format & Content

### 7.1 Receipt Structure

**The receipt must include the following elements in this order:**

1. **Custom Letterhead**
   - Business can upload their own letterhead via Store Manager settings
   - Displayed at the top of every receipt

2. **Receipt Number**
   - Auto-generated unique number
   - Format: Alphanumeric (e.g., RCP-2024-A001, RCP-001-B)
   - Must be unique for each sale
   - Increments with each sale in the system

3. **Customer Information**
   - Customer name
   - Phone number

4. **Item Details** (For each item sold)
   - Item name
   - SKU
   - Quantity sold
   - Unit price
   - Subtotal (Quantity × Unit Price)

5. **Total Amount Paid**
   - Grand total of the sale

6. **Location Information**
   - Store/location name where sale was made

7. **Date & Time**
   - Date of transaction
   - Time of transaction

### 7.2 Exclusions

- **NO employee/staff name** on receipt

### 7.3 Export Options

- **Print:** Direct printing to connected printer
- **Save as PDF:** Save receipt file on device for later use or email

## 8. Returns & Reversal Process (Business Admin Only)

### 8.1 Access Control
- **Only Business Admins** can access and process returns
- Regular employees cannot access this function

### 8.2 Search Methods

Business Admins can search for sales using any of the following methods:

#### Method 1: By Customer Name
- Enter customer name
- System displays **all sales** made to that customer
- Shows sale history across all transactions

#### Method 2: By Receipt Number
- Enter receipt number
- System displays **specific items** from that receipt
- Shows only the items in that particular transaction

#### Method 3: By Phone Number
- Enter customer phone number
- System displays **all items** sold to that customer
- Shows complete purchase history

### 8.3 Return Process

1. **Search** for the sale using one of the three methods above
2. Sale details appear showing all items in that transaction
3. Click **"Reverse Sale"** button
4. System prompts: **"Select items to return"**
5. Check which items are being returned (can select one or multiple items from that sale)
6. Confirm selection
7. **Inventory automatically increases back** by the returned quantity
8. Return is **recorded as a separate transaction** in the system:
   - Maintains audit trail
   - Shows original sale + reversal transaction
   - Not just a simple undo

### 8.4 Return Transaction Record

- Original sale remains in system (marked as partially/fully returned)
- Return transaction is created separately
- Both transactions are linked for traceability
- Useful for auditing, reporting, and tracking returns

## 9. Sold Status View in Inventory

### 9.1 Viewing Sold Items

When an item is marked as **"Sold"** in the inventory list:

#### For All Users (Cashiers, Managers, etc.):
- Click on the **"Sold"** status
- Display shows: **Customer ID only**
- Cannot view:
  - Customer name
  - Phone number
  - Full transaction details

#### For Business Admins Only:
- Click on the **"Sold"** status
- Can view:
  - **Customer ID**
  - **Customer name**
  - **Phone number**
  - **Full transaction details**
  - **Receipt number**
  - **All customer information**
- Can access and process returns from this view

### 9.2 Purpose

This privacy control ensures:
- Regular employees cannot access customer personal information
- Data security and privacy compliance
- Business Admins have full visibility for management and returns
- Audit trail is maintained

## 10. Data Structure - Customer ID

### 10.1 Customer ID Generation
- Automatically generated when a new customer is created
- Unique identifier for each customer in the system
- Used in sold items view for privacy (shows ID instead of name)

### 10.2 Usage
- Displayed in inventory when items are sold (for non-admin users)
- Business admins can link Customer ID to full customer information
- Used in return searches and transaction lookups

## 11. Integration Points with Scanner App

1. **Dynamic QR Code Linking:** Scanner scans the QR code from Setup Scanner in Store Manager App
2. **Scan Commands:** Store Manager App sends scan commands to Scanner App for:
   - QR code tracking in inventory
   - QR code scanning during sales
3. **Data Return:** Scanner App returns only the **QR code value** to Store Manager App
4. **User Context:** Scanner knows which user is logged in (for accountability and location context)
5. **Session Management:** Scanner maintains session until manual logout

---

## 12. Implementation Notes

- All inventory updates are backend operations
- Customer IDs are system-generated and immutable
- Receipt numbers must be unique and sequential
- Letterhead upload functionality must validate file types and dimensions
- All transactions must include timestamp for audit purposes
- Sold items with "Pending Sale" status should be visually distinct from "Sold" items
- Return transactions create permanent records for compliance
